Question 1
A GenAI chatbot receives user queries that often contain typos, extra whitespace, special characters, and inconsistent capitalization. Before sending the query to Amazon Bedrock, the development team wants to clean and normalize the text to improve the quality of the foundation model's responses.
Which AWS service should the developer use to implement this text pre-processing logic?
Amazon Comprehend
AWS Lambda
AWS Glue
Amazon Textract
CORRECT: AWS Lambda
AWS Lambda is ideal for running lightweight data transformation and pre-processing logic. The developer can write a Lambda function (in Python, Node.js, etc.) to normalize text by removing extra whitespace, correcting common typos, standardizing capitalization, and sanitizing special characters before passing the cleaned text to Bedrock. Lambda's serverless nature means you only pay for the milliseconds of compute time used for each pre-processing operation.
INCORRECT: Amazon Comprehend
Amazon Comprehend is a natural language processing service that extracts insights from text (entities, sentiment, key phrases, PII). While Comprehend analyzes text, it does not perform text normalization, cleaning, or transformation. Comprehend tells you
what's in the text, but doesn't modify it. You would use Lambda to clean the text, then optionally use Comprehend for analysis.
INCORRECT: AWS Glue
AWS Glue is a serverless data integration service designed for ETL (Extract, Transform, Load) workflows on large datasets, typically for analytics and data warehousing. Glue is optimized for batch processing of big data, not for low-latency, per-request text cleaning in real-time chatbot interactions. Lambda is the right choice for lightweight request/response transformations.
INCORRECT: Amazon Textract
Amazon Textract extracts text from scanned documents and images (OCR). It converts visual documents into digital text but does not clean or normalize already-digital text input from users. Textract is for digitization, not for text preprocessing or normalization of user queries.
Key Concepts:
Lambda = serverless compute for data transformation and pre-processing
Text normalization = removing typos, extra spaces, standardizing format
Real-time processing = Lambda provides low-latency request/response transformation
